# n8n Workflow Inventory

Scan all local `.workflow.ts` files, aggregate usage statistics, and write a structured report to `docs/INVENTORY.md`.

> **Note:** This skill only reads files that are already checked out locally. If workflows are missing, run `npx n8nac pull <id>` first, or `npx n8nac list --json` to see which IDs exist remotely.

## Prerequisites

- n8nac must be initialized (`npx n8nac init`)
- At least one `.workflow.ts` file must exist under `workflows/`

## Steps

### 1. Locate Workflow Files

```bash
find workflows -name "*.workflow.ts" -type f 2>/dev/null | sort
```

If the output is empty: stop and report — no local workflow files found. Tell the user to run `npx n8nac pull <id>` for each workflow ID visible in `npx n8nac list --json`.

### 2. Fetch Remote Metadata (active / archived counts)

```bash
npx n8nac list --json 2>/dev/null
```

Parse the JSON array to get:
- Total remote workflows
- Count where `active: true`
- Count where `isArchived: true`

This is only for the Summary header — skip gracefully if n8nac is unreachable.

### 3. Extract and Aggregate

Run the following grep commands across **all** discovered workflow files (pass all paths at once, not in a loop):

**Node Types** (only match real n8n package prefixes to avoid TypeScript false-positives; includes scoped community packages like `@mendable/`):
```bash
find workflows -name "*.workflow.ts" -print0 | xargs -0 \
  grep -hE "^\s+type: '(@[a-zA-Z][^/]+/n8n-nodes-[^']+|n8n-nodes-base\.[^']+|n8n-nodes-[^']+)'" \
  | grep -oE "'[^']+'" | tr -d "'" | sort | uniq -c | sort -rn
```

**Credential Types** (single-line inline pattern — how n8nac generates credentials):
```bash
find workflows -name "*.workflow.ts" -print0 | xargs -0 \
  grep -hoE "credentials: \{ [a-zA-Z][a-zA-Z0-9]+:" \
  | grep -oE "[a-zA-Z][a-zA-Z0-9]+:" | grep -v "^credentials:" | tr -d ':' \
  | sort | uniq -c | sort -rn
```

**LLM Models** (matches `value: 'gpt-...'`, `value: 'claude-...'`, `value: 'llama...'`, `value: 'gemini...'`, `value: 'mistral...'`):
```bash
find workflows -name "*.workflow.ts" -print0 | xargs -0 \
  grep -hoE "value: '(gpt|claude|llama|gemini|mistral|o[0-9])[^']+'" \
  | grep -oE "'[^']*" | tr -d "'" | sort | uniq -c | sort -rn
```

**Workflow names per credential type** (for the Credentials table "Example Workflows" column):
```bash
# For each credentialType, find which workflow files reference it:
find workflows -name "*.workflow.ts" -print0 | xargs -0 \
  grep -lE "credentials: \{ <credType>:"
# Replace <credType> with each credential key found in step above.
# Extract the workflow name from the @workflow({ name: '...' }) decorator or filename.
```

### 4. Classify and Structure Results

From the node-type counts, derive:

- **Core nodes:** `type` starts with `n8n-nodes-base.` → strip prefix, keep node name
- **Community nodes:** everything else (`@n8n/n8n-nodes-langchain.*`, `n8n-nodes-*`, etc.)
- **Trigger nodes:** node names matching `.*[Tt]rigger|webhook|schedule|chatTrigger|formTrigger|mcpTrigger|manualTrigger|errorTrigger`
- **Utility-only nodes to flag:** `stickyNote` (cosmetic, not logic — note count but exclude from "active logic nodes" count)

For LLM models, infer the provider from the node type:
- `lmChatOpenAi` → **OpenAI**
- `lmChatAnthropic` → **Anthropic**
- `lmChatOllama` → **Ollama**
- `lmChatGoogleGemini` / `lmChatGoogleVertex` → **Google**
- `lmChatMistralCloud` → **Mistral**
- Others → **Other**

For credential type names, map common suffixes to friendly labels where obvious (e.g. `openAiApi` → OpenAI API, `googleDriveOAuth2Api` → Google Drive OAuth2) — or just show the raw key name.

### 5. Write Report

If `--dry-run` flag is set: print the report to stdout only. Otherwise write to `docs/INVENTORY.md`.

Report structure:

```markdown
# n8n Workflow Inventory

> Auto-generated by `/n8n-autopilot:inventory` on YYYY-MM-DD. Do not edit by hand.

## Summary

| Metric | Count |
|---|---|
| Local workflow files | N |
| Remote workflows (total) | N |
| Active (remote) | N |
| Archived (remote) | N |

## Trigger Distribution

| Trigger Type | Count |
|---|---|
| chatTrigger | 4 |
| webhook | 3 |
| schedule | 2 |

## Node Usage — Core (`n8n-nodes-base`)

Top 10 by usage count, sorted descending.

| Node | Count |
|---|---|
| code | 47 |
| set | 23 |
| ... | ... |

## Node Usage — Community

| Package | Node | Count |
|---|---|---|
| @n8n/n8n-nodes-langchain | agent | 12 |
| @n8n/n8n-nodes-langchain | lmChatOpenAi | 9 |
| n8n-nodes-firecrawl | firecrawl | 4 |

## LLM Models

| Provider | Model | Count |
|---|---|---|
| OpenAI | gpt-4o-mini | 6 |
| OpenAI | gpt-4.1-mini | 3 |
| Anthropic | claude-sonnet-4-6 | 2 |

## Credentials

| Credential Type | Count | Example Workflows (max 3) |
|---|---|---|
| openAiApi | 8 | Angebotsagent, Inbox Copilot, … |
| googleDriveOAuth2Api | 3 | Angebotsagent, … |
```

If `--json` flag is set: additionally write `docs/inventory.json` with the same data in structured form:

```json
{
  "generatedAt": "YYYY-MM-DD",
  "summary": { "localFiles": N, "remoteTotal": N, "active": N, "archived": N },
  "triggers": { "chatTrigger": 4, "webhook": 3 },
  "coreNodes": { "code": 47, "set": 23 },
  "communityNodes": { "@n8n/n8n-nodes-langchain.agent": 12 },
  "llmModels": { "OpenAI/gpt-4o-mini": 6 },
  "credentials": { "openAiApi": 8 }
}
```

### 6. Completion Report

Print a brief summary to the console:

```
✓ Inventory complete
  Local files scanned : 11
  Unique node types   : 34 (22 core, 12 community)
  LLM models found    : 3
  Credential types    : 7
  Report written to   : docs/INVENTORY.md
```

## Error Handling

- **No `.workflow.ts` files found** → stop, tell user to run `npx n8nac pull <id>`
- **n8nac unreachable** → skip remote metadata section, note it in Summary with "N/A"
- **`docs/` directory doesn't exist** → create it before writing

## Example Invocations

```
/n8n-autopilot:inventory                  # Full scan + write docs/INVENTORY.md
/n8n-autopilot:inventory --dry-run        # Print report to stdout, no files written
/n8n-autopilot:inventory --json           # Also write docs/inventory.json
/n8n-autopilot:inventory --dry-run --json # Preview both outputs, no files written
```
